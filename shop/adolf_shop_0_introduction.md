## Что такое ADOLF SHOP

ADOLF SHOP — функциональный модуль системы ADOLF, обеспечивающий AI-управление интернет-магазином на базе WordPress/WooCommerce через протокол MCP (Model Context Protocol).

### Интеграция с WooCommerce

| Параметр | Значение |
|----------|----------|
| Платформа | WordPress + WooCommerce 10.3+ |
| Протокол | MCP (Model Context Protocol) |
| Магазин | ohana.market |
| Хостинг | Timeweb VPS |

### Суббренды магазина

| Бренд | Описание |
|-------|----------|
| Охана Kids | Детская одежда |
| Охана Style | Взрослая одежда |
| Охана Care | Товары для ухода |
| Охана Lux | Премиум-сегмент |

Суббренды реализованы через плагин WooCommerce Brands (таксономия `product_brand`).

### Основные возможности v1.0

| Функция | Описание |
|---------|----------|
| Мониторинг заказов | Отслеживание новых заказов, отмен, возвратов |
| Управление заказами | Просмотр деталей, поиск, фильтрация |
| Поиск товаров | Поиск по каталогу через MCP |
| Аналитика продаж | Выручка, средний чек, топ товаров |
| Уведомления | Push-уведомления о событиях через Notifications |

## Функциональный скоуп

### MVP (v1.0)

**Заказы:**
- Сигнал о новом заказе
- Сигнал об отмене заказа
- Сигнал о возврате
- Заказы в ожидании (pending)
- Проблемные заказы (failed, refunded)
- Поиск заказов
- Просмотр деталей заказа

**Каталог:**
- Поиск товара по запросу

**Аналитика:**
- Выручка (за период, по брендам)
- Количество заказов
- Средний чек
- Топ товаров по продажам
- Продажи по брендам
- Заказы без оплаты (> 48 часов)

## Архитектура интеграции

```
┌─────────────────┐     MCP      ┌──────────────────┐
│   Open WebUI    │◄────────────►│   WooCommerce    │
│   + Claude      │              │   MCP Server     │
└────────┬────────┘              └────────┬─────────┘
         │                                │
         │                                │
┌────────▼────────┐              ┌────────▼─────────┐
│  ADOLF Core     │              │   WordPress      │
│  (Celery,       │              │   ohana.market   │
│   Notifications)│              │                  │
└─────────────────┘              └──────────────────┘
```

**Принцип работы:**
1. Claude в Open WebUI получает доступ к WooCommerce MCP tools
2. Пользователь задаёт вопрос через чат
3. Claude использует MCP для получения данных из WooCommerce
4. Celery Beat периодически проверяет новые события
5. При обнаружении события — push через Notifications

## Быстрый старт

### Для пользователя (Senior+)

1. Откройте Open WebUI
2. Выберите агента `@Adolf_Shop`
3. Примеры запросов:
   - "Покажи новые заказы"
   - "Найди заказ #12345"
   - "Аналитика продаж за неделю"
   - "Топ-10 товаров по продажам"
   - "Заказы в ожидании оплаты"

### Для администратора

1. Убедитесь, что WooCommerce MCP активирован
2. Проверьте Celery Beat: `celery -A app beat`
3. Проверьте Celery Worker: `celery -A app worker`
4. Проверьте логи polling в Redis
