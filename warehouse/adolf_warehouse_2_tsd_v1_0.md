# ADOLF WAREHOUSE — Раздел 2: ТСД

**Проект:** Управление физическим складом  
**Модуль:** Warehouse / TSD  
**Версия:** 1.0 (черновик)  
**Дата:** Май 2026

---

## 2.1 Назначение

Раздел описывает архитектуру ТСД-режима — отдельной мобильной страницы `/warehouse/tsd`, оптимизированной под работу кладовщика на бюджетном Android со сканером.

---

## 2.2 Ключевые требования

| Требование | Метрика |
|------------|---------|
| Скорость отклика на скан | < 100 мс от bip до зелёного на экране |
| Offline-устойчивость | 100% операций без сети, синк при возврате |
| Bundle size | < 200 КБ JS на главный экран |
| Минимальная версия Android | 8.0 (Chromium 70+) |
| Touch targets | ≥ 44 px |

---

## 2.3 Авторизация

ТСД-операторы **не имеют ADOLF-аккаунтов** — отдельная таблица `wh_tsd_operators` с ПИН-кодами.

Вход: ПИН (4-6 цифр) + опционально IP-allowlist (только из подсети склада).

При неактивности > 30 минут — авто-логаут.

---

## 2.4 Подключение сканера

### Поддерживаемые способы

| Способ | Описание | Когда использовать |
|--------|----------|---------------------|
| **HID-keyboard mode** | Сканер эмулирует ввод символов | Самый совместимый, любой Android |
| **Web Serial API** | USB OTG-сканер через serial | Стационарные ТСД |
| **WebHID API** | Низкоуровневое подключение | Bluetooth-браслеты |
| **Bluetooth SPP** | Через нативную обёртку (Capacitor) | Если нужен полный нативный доступ |

Автоопределение при старте: сначала пробуем WebHID, потом Web Serial, fallback на keyboard-mode.

## 2.5 Offline-архитектура

### IndexedDB схема (через Dexie.js)

| Таблица IDB | Содержимое |
|-------------|------------|
| `outbox_events` | Накопленные события сканирования (для синка) |
| `cache_nomenclature` | Snapshot справочника GTIN → артикул (10-30 МБ) |
| `cache_active_doc` | Текущий открытый документ + план |
| `local_drafts` | Черновики QC-фото (до загрузки) |

### Service Worker

- Кэшируется весь shell (HTML/JS/CSS/иконки) для оффлайн-старта
- Сетевой strategy: **stale-while-revalidate** для UI, **network-first** для данных
- Background Sync API для отправки outbox при появлении сети

## 2.6 Звук и тактильная обратная связь

### Web Audio API — генерация бипа

Не загружаем .mp3 (latency, IO), генерим тон через `OscillatorNode`:

```js
const ctx = new AudioContext();
const beep = (freq = 800, duration = 60) => {
  const osc = ctx.createOscillator();
  const gain = ctx.createGain();
  osc.frequency.value = freq;
  osc.connect(gain).connect(ctx.destination);
  osc.start();
  gain.gain.setValueAtTime(0.2, ctx.currentTime);
  gain.gain.exponentialRampToValueAtTime(0.01, ctx.currentTime + duration / 1000);
  osc.stop(ctx.currentTime + duration / 1000);
};
```

Палитра звуков:
- Успех: 800 Hz, 60 мс
- Ошибка: 200 Hz, 200 мс (низкий)
- Дубль: два коротких 200 Hz подряд

### Vibration API

`navigator.vibrate(pattern)` для:
- Успех: 50 мс
- Ошибка: 200 мс
- Дубль: [100, 50, 100]

## 2.7 UX-паттерны

### Optimistic UI

Любое сканирование сразу обновляет счётчик локально, без ожидания сервера. Если сервер потом отбросил — visual rollback с пояснением.

### Минимум tap'ов

Главная страница ТСД — это просто счётчик и две большие кнопки (Пауза / Закончить). Всё остальное — через сканирование физических меток (ячейки, контейнеры, маршрутные листы).

### Большие шрифты

База: 18px. Цифра прогресса: 96px. Padding кнопок: 16px+.

## 2.8 Режимы работы

| Режим | Триггер запуска | Цель |
|-------|-----------------|------|
| Приёмка | Список приходов в статусе `planned` | Оприходовать товар |
| ОТК | Список партий ожидающих ОТК | Проверить качество |
| Размещение | Партии после ОТК | Развезти по ячейкам |
| Комплектация | Назначенные заказы | Собрать заказ клиенту |
| Фасовка | Собранные заказы | Запаковать |
| Отгрузка | Маршруты отгрузки | Передать перевозчику |
| Инвентаризация | Назначенные задания | Пересчитать ячейки |
| Перемещение | По требованию | Внутри склада |

## 2.9 Безопасность ТСД

| Мера | Реализация |
|------|------------|
| ПИН-код | bcrypt в `wh_tsd_operators.pin_hash` |
| IP-allowlist | Только подсеть склада (опционально) |
| Авто-логаут | После N мин простоя |
| Audit-trail | Каждое действие → `wh_movements.user_id = tsd_operator_id` |
| Защита от подмены ТСД | Device fingerprint (UUID + сертификат) |
