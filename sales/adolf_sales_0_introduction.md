## 0.1 Назначение

Модуль **Sales** — это полностью автоматизированная замена ручной расчётно-аналитической таблицы Google Sheets («туфли реклама»), которой пользовались менеджеры до внедрения ADOLF. Модуль ведёт операционную аналитику продаж в разрезе каждого SKU на всех трёх маркетплейсах и сравнивает факт с планом на уровне дня, недели и месяца.

Модуль закрывает два контура:

| Контур | Что считает |
|--------|-------------|
| **Юнит-экономика** | По каждому SKU: цена, штуки, комиссия, логистика, хранение, реклама, ДРР, налог, себестоимость, маржа, маржинальность, ROI, % выкупа |
| **План-факт** | Сопоставление план/факт по дню, неделям 1–4 и месяцу; сводка «сколько SKU выполнили план»; трекер темпа месяца; недельные отчёты в разрезе ABC-категорий и новинок |

<Note>
**Главный принцип модуля — нулевой ручной ввод.** Старая таблица требовала ежедневной ручной вставки 5–8 выгрузок из личных кабинетов. Модуль Sales собирает все данные через API маркетплейсов и из 1С автоматически по расписанию Celery.
</Note>

## 0.2 Место в платформе ADOLF

Модуль Sales опирается на уже существующие компоненты платформы и не дублирует их:

| Компонент | Что переиспользуется |
|-----------|----------------------|
| **1Cexport** | Себестоимость по SKU из `1C_account_turns_90` (счёт 90.02.1) и закупочные цены из `1C_nomenclature_prices`; бренд из `1C_nomenclature.kind` |
| **CFO** | VIEW `cfo_v_cost_prices` (готовая себестоимость по артикулу); адаптеры авторизации маркетплейсов |
| **Logistic** | Конфигурация и паттерн интеграции Ozon Seller API (`OzonAPISettings`) |
| **Core** | PostgreSQL, FastAPI middleware, Celery + Redis, Open WebUI, система ролей и брендов |

**Отличие от CFO.** CFO формирует управленческий P&L и финансовую отчётность (бухгалтерский контур, еженедельно/ежемесячно). Sales — операционный контур: ежедневная аналитика по SKU и контроль выполнения плана менеджерами. Источники частично общие, потребитель и периодичность — разные.

## 0.3 Источники vs ручной ввод

Каждый показатель старой таблицы имеет автоматический источник. Ручной ввод устранён полностью:

| Данные старой таблицы | Было | Стало (источник) |
|-----------------------|------|------------------|
| Финансовый отчёт | Excel из ЛК | WB `reportDetailByPeriod` / Ozon `finance/transaction/list` / YM `stats/orders` |
| Продажи и заказы | Excel из ЛК | WB `sales` + воронка / Ozon `analytics/data` / YM `stats/skus` |
| Воронка (корзина, выкуп) | Excel из ЛК | WB `nm-report/detail` / Ozon `analytics/data` / YM `stats/skus` |
| Цены и скидки | Excel из ЛК | WB `list/goods/filter` / Ozon `product/info/prices` / YM offer-mappings |
| Остатки | Excel из ЛК | WB `stocks` / Ozon `product/info/stocks` / YM stats |
| Реклама | Excel из ЛК | WB `adv/v2/fullstats` / Ozon Performance API / YM bids |
| Комиссия по категориям | Ручной справочник | WB `tariffs/commission` / факт из финотчётов |
| Себестоимость | Ручной справочник | **1С:** `cfo_v_cost_prices` / `1C_account_turns_90` |
| План продаж | Ручной ввод | **Авто-прогноз** (см. Раздел 4) или план из 1С |
| Налоговая ставка | Ручная константа | Конфиг модуля (НДС 22% с 2026, см. Раздел 3) |

## 0.5 Маркетплейсы

Модуль работает с тремя площадками в едином нормализованном представлении:

| Маркетплейс | Base URL | Авторизация |
|-------------|----------|-------------|
| Wildberries | `https://*-api.wildberries.ru` | JWT-токен (заголовок `Authorization`) |
| Ozon | `https://api-seller.ozon.ru` | `Client-Id` + `Api-Key` |
| Яндекс.Маркет | `https://api.partner.market.yandex.ru` | API-Key токен; `businessId` + `campaignId` |
