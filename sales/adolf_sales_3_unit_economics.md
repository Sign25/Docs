Расчёт повторяет логику листа «Юнит Факт» исходной таблицы, но источники — нормализованный `sales_fact_daily` (Раздел 2.5) и себестоимость из 1С. Все блоки считаются по каждому SKU за выбранный период и складываются в `sales_unit_economics`.

## 3.1 Объёмные показатели

| Блок | Формула | Пояснение |
|------|---------|-----------|
| Чистые продажи, шт (`net_qty`) | `buyouts_qty − returns_qty` | База для всех удельных показателей |
| Средняя цена продажи (`price_avg`) | `(Σ buyouts_sum − Σ returns_sum) / net_qty` | Фактическая цена нетто после всех скидок; при отсутствии продаж — текущая цена из блока цен |
| Выручка (`revenue`) | `net_qty × price_avg` | |

## 3.2 Затраты (факт из API)

| Блок | Источник | Формула |
|------|----------|---------|
| Комиссия | финотчёт / `tariffs/commission` | `Σ commission` (факт) либо `price_avg × ставка%` |
| Логистика | финотчёт | `Σ logistics` |
| Хранение | финотчёт | `Σ storage` (по SKU, без пропорционального разноса) |
| Штрафы | финотчёт | `Σ penalty` |
| Реклама | `adv/v2/fullstats` / Performance / bids | `Σ ad_spend` |
| ДРР (`drr`) | расчёт | `ad_spend / revenue` |

<Note>
В отличие от старой таблицы, хранение и приёмка берутся фактом по SKU из финансовых отчётов площадок, а реклама — прямым расходом по идентификатору товара. Пропорциональный разнос по объёму и привязка рекламы по названию кампании больше не нужны.
</Note>

## 3.3 Себестоимость

| Блок | Источник | Формула |
|------|----------|---------|
| Себестоимость единицы (`cogs_unit`) | `sales_v_cost_prices` (из 1С) | lookup по SKU |
| Себестоимость общая (`cogs`) | расчёт | `net_qty × cogs_unit` |

## 3.4 Налог (НДС)

<Warning>
Исходная таблица считала налог упрощённо как «процент от цены» (модель УСН). Для ООО «Охана Маркет» это **неверно** — компания на НДС. Блок переписан.
</Warning>

С 1 января 2026 г. базовая ставка — **НДС 22%** (`sales_settings.vat_rate = 0.22`). Налог считается не от цены, а как разница исходящего и входящего НДС:

```
НДС_исходящий = revenue × vat_rate / (1 + vat_rate)
НДС_входящий  = cogs    × vat_rate / (1 + vat_rate)     # при поставщиках-плательщиках НДС
НДС_к_уплате  = НДС_исходящий − НДС_входящий
```

Если поставщик не плательщик НДС, входящий НДС = 0 (вычета нет). Бенчмарк чистой маржи для ОКВЭД 46.42 (оптовая одежда) под НДС 22% с вычетами — ориентир 3–5% от выручки; модуль рассчитывает фактическое значение, а не подставляет бенчмарк.

## 3.5 Маржа и эффективность

| Блок | Формула | Пояснение |
|------|---------|-----------|
| Маржа (`margin`) | `revenue − commission − logistics − storage − penalty − ad_spend − НДС_к_уплате − cogs` | Чистая прибыль по SKU |
| Маржа без рекламы | `margin + ad_spend` | Органическая рентабельность |
| Маржа на 1 продажу | `margin / net_qty` | |
| Маржинальность (`margin_pct`) | `margin / revenue` | |
| ROI (`roi`) | `(margin / net_qty) / cogs_unit` | Прибыль на рубль себестоимости |
| % выкупа (`buyout_pct`) | из воронки (`buyoutPercent`) или `buyouts_qty / orders_qty` | |

## 3.6 ABC-анализ

Категория присваивается по доле SKU в метрике (выручка / маржа) относительно средней доли, пороги — из `sales_settings.abc_thresholds`:

| Условие | Категория |
|---------|-----------|
| доля > средняя × 1.5 | A |
| доля > средняя × 1.0 | B |
| иначе | C |

Категория пишется в `sales_unit_economics.abc_category` и используется в недельных отчётах план-факта (Раздел 4.4).
